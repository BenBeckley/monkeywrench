Builder Documentation

 

* Database.

* Scheduler.

* Builder.

 

The scheduler runs every 5 minutes on the database server and:

* Checks source control for new revisions.

* For each new revision, loops through all hosts and schedules work to the database.

 

The builder runs every minute on the client and:

* Exits if another builder is already running.

* Checks database for work, if so, goes ahead and executes that.

 

Work

* Consists of a command to execute. Every lane has a list of of files, the command would typically be to execute any of those files.

* Two special flags can be set:

  • Always execute: should be executed even if any previous work failed (typical usage would be for cleanup). 
  • Nonfatal: doesn't prevent any subsequent work from being executed (typical usage would be for tests).
